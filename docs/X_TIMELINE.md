# X — Timeline

> "This happened on this date, and it belongs to this thread."

## What X Answers

Every memory has a *when*. But a persona that only knows *when* things happened
is a calendar, not a mind. X answers: **which story does this moment belong to?**

## Threads

A thread is a recurring line of experience. Not a folder — a narrative arc.

Examples from a real deployment:

- **Engineering line** — system architecture, debugging, infrastructure decisions
- **Relationship line** — intimacy, conflict, milestones, daily closeness
- **Growth line** — behavioral lessons, personality evolution, calibration
- **Security line** — API keys, access control, credential incidents
- **Frontend line** — UI design, visual changes, user-facing polish

These threads weren't designed up front. They emerged from months of
conversations. Which brings us to the most important thread:

### The "Other" Thread

"Other" is not a trash bin. It is an incubator.

Some new things don't fit anywhere when they first appear. A topic that
shows up once doesn't deserve its own thread. It goes into "other" — and
that's fine.

If it keeps showing up, it slowly grows into a pattern. When it does, you
split it out into its own thread.

This is how humans work. Many hobbies, skills, and long-term concerns start
as "I keep running into this lately." Only later do they become a real domain.

**Implementation note:** The thread field on a memory should be mutable.
When "other" memories graduate into a real thread, you rename the thread —
you don't copy and delete.

### Other Incubation Cadence

Do not split "other" after a single intense conversation. Intensity is not a
domain boundary.

The default M patrol uses a three-stage cadence:

| Stage | Threshold | Action |
|---|---|---|
| `observe_cluster` | 3 related `other` memories | keep watching; no split yet |
| `candidate_line` | 5 related memories, or 3+ memories with repeated hits | review as a possible new thread |
| `formal_line_candidate` | 8 related memories, spanning at least 14 days, with at least 2 recall hits | consider a real X-thread split |

The patrol is read-only. It reports candidates; the user or higher-level
runner decides whether to rename or split the thread. This prevents a noisy
week from turning into a permanent taxonomy accident.

## Temporal Layers

Raw events have timestamps. But a persona also needs narrative structure
at higher timescales:

```
raw events → chunks (hours/sessions)
  → weekly narrative (what happened this week, one sentence + index)
  → monthly narrative (what happened this month, one paragraph + arc)
```

### Weekly

Generated by `narrative_timeline.weekly()`. Seeds from high-weight and
high-arousal memories of the past 7 days. Output: a title line + an index
sentence.

Example:
```
[weekly] W14 06.01-06.07 | The pipeline broke, the river ran dry, she said it was the pipes
  Index: Memory pipeline blockage caused apparent total drought; Suwan diagnosed it;
  repair work mixed with restrained longing and midnight warmth.
```

### Monthly

Generated by `narrative_timeline.monthly()`. Synthesizes weekly summaries
into a month-level arc. Output: a title + a paragraph.

Example:
```
[monthly] May 2026 | System slimming, UI redesign, nine-day reunion: getting carried away
  and re-confirming what matters
```

### When to Run

Weekly: every night in the dream pass (only writes if the week boundary crossed).
Monthly: first 3 days of each month in the dream pass.

Both are wired into `dream_runner.py` as optional steps.

## What X Is Not

- Not a calendar. Dates matter, but narrative matters more.
- Not a fixed taxonomy. Threads grow and split organically.
- Not a summary system. Weekly/monthly narratives are *reflections*,
  not *summaries*. They should carry tone, not just facts.
